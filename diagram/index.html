rm-viewer Sync Architecture

Triple-buffer pipeline with syncd reconcile loop

Happy Path — Full Sync Cycle

sequenceDiagram
    participant X as xochitl
(device fs)
    participant S as sync.sh
(device)
    participant D as xochitl-dirty
(remote)
    participant SY as syncd
(remote)
    participant ST as xochitl-staging
(remote)
    participant XR as xochitl
(remote)

    X->>S: inotify event
    Note over S: debounce 30s
    Note over S: flock acquired

    S->>S: SSH: syncflag exists? NO
    S->>D: rsync (--contimeout=30, --timeout=60)

    S->>D: create syncflag
    Note over S: release flock

    Note over SY: wakes: syncflag event

    Note over SY: reconcile #1 — stage
    SY->>SY: syncflag? YES. staging_full? NO
    D->>ST: rsync dirty → staging
    SY->>D: delete syncflag
    Note over SY: staging_full = True

    Note over SY: reconcile #2 — promote + process
    SY->>SY: staging_full? YES. processing? NO
    ST->>XR: rsync staging → xochitl
    SY->>XR: spawn rm-process
    Note over SY: processing = True
staging_full = False

    Note over SY: reconcile #3 — nothing to do, sleep

    Note over SY: wakes: child exit
    Note over SY: reconcile — reap
    SY->>SY: rm-process exited (success)
    SY->>XR: trigger viewer rebuild
    Note over SY: processing = False

Stall Path — rsync Fails, Next Cycle Resumes

sequenceDiagram
    participant X as xochitl
(device fs)
    participant S as sync.sh
(device)
    participant D as xochitl-dirty
(remote)

    X->>S: inotify event
    Note over S: debounce 30s, flock acquired
    S->>S: SSH: syncflag exists? NO
    S->>D: rsync (--contimeout=30, --timeout=60)
    Note over S,D: connection stalls after 60s inactivity
    Note over S: rsync fails, no syncflag created
    Note over S: release flock

    X->>S: new inotify event
    Note over S: debounce 30s, flock acquired
    S->>S: SSH: syncflag exists? NO
    S->>D: rsync (resumes via --partial-dir, completes)
    S->>D: create syncflag
    Note over S: release flock

Blocked Path — Device Skips When syncflag Present

    S->>S: SSH: syncflag exists? NO
    S->>D: rsync (--contimeout=30, --timeout=60)

    S->>D: create syncflag
    Note over S: release flock
    Note over SY: syncd wakes, reconcile...

    Note over S: new inotify event, debounce, flock
    S->>S: SSH: syncflag exists? YES
    Note over S: skip, release flock

    Note over S: another inotify event, debounce, flock
    S->>S: SSH: syncflag exists? YES
    Note over S: skip, release flock

    Note over SY: syncd deletes syncflag

    Note over S: another inotify event, debounce, flock
    S->>S: SSH: syncflag exists? NO
    S->>D: rsync (proceeds normally)

Concurrent — Multiple Syncs During Slow rm-process

sequenceDiagram
    participant S as sync.sh
(device)
    participant D as xochitl-dirty
(remote)
    participant SY as syncd
(remote)
    participant ST as xochitl-staging
(remote)
    participant XR as xochitl
(remote)

    Note over XR: rm-process running (slow)

    S->>D: sync #2 completes, create syncflag
    Note over SY: wakes: syncflag event
    Note over SY: reconcile — reap: processing, not exited
    SY->>SY: syncflag? YES. staging_full? NO
    D->>ST: rsync dirty → staging
    SY->>D: delete syncflag
    Note over SY: staging_full = True
    SY->>SY: staging_full? YES. processing? YES
    Note over SY: skip promote, sleep

    S->>D: sync #3 completes, create syncflag
    Note over SY: wakes: syncflag event
    SY->>SY: syncflag? YES. staging_full? YES
    Note over SY: skip stage (data sits in dirty), sleep

    Note over XR: rm-process finishes
    Note over SY: wakes: child exit
    SY->>SY: reap rm-process (success)
    SY->>XR: trigger viewer rebuild
    Note over SY: processing = False

    SY->>SY: staging_full? YES. processing? NO
    ST->>XR: rsync staging → xochitl
    SY->>XR: spawn rm-process
    Note over SY: processing = True, staging_full = False

    SY->>SY: syncflag? YES. staging_full? NO
    D->>ST: rsync dirty → staging (latest)
    SY->>D: delete syncflag
    Note over SY: staging_full = True

    SY->>SY: staging_full? YES. processing? YES
    Note over SY: skip promote, sleep
(pipeline drains...)
